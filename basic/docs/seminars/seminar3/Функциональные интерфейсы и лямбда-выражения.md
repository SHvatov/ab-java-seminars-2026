# Функциональные интерфейсы и лямбда-выражения

---

## Функциональный интерфейс

Интерфейс ровно с одним абстрактным методом (SAM — *single abstract method*). `default`- и `static`-методы не
считаются — их может быть сколько угодно.

```java
public interface InstrumentFilter {
    boolean matches(Instrument instrument);
}
```

Один абстрактный метод — интерфейс уже функциональный, даже без аннотации. `@FunctionalInterface` ничего не меняет в
поведении — она лишь просит компилятор проверить, что интерфейс действительно SAM, и упасть с ошибкой, если кто-то
случайно добавит второй абстрактный метод.

---

## Лямбда-выражение

Компактная реализация функционального интерфейса прямо в месте использования, без отдельного класса:

```java
// было: отдельный класс на каждую стратегию
InstrumentFilter filter = new TypeFilter("STOCK");

// стало: лямбда вместо класса
InstrumentFilter filter = instrument -> instrument.getClass().getSimpleName().equalsIgnoreCase("STOCK");
```

Синтаксис: `(параметры) -> выражение` или `(параметры) -> { блок кода }`. Тип параметров выводится компилятором из
контекста (из сигнатуры абстрактного метода), поэтому параметр обычно пишется без явного типа.

---

## Захват переменных

Лямбда может обращаться к локальным переменным окружающего метода — это называется *захватом* (capture):

```java
void example() {
    String type = "STOCK";
    InstrumentFilter filter = instrument -> instrument.getClass().getSimpleName().equalsIgnoreCase(type);
}
```

Условие: переменная должна быть *effectively final* — фактически неизменяемой, то есть ей нельзя присваивать значение
повторно после инициализации (даже если явного `final` нет).

```java
void example() {
    String type = "STOCK";
    InstrumentFilter filter = instrument -> instrument.getClass().getSimpleName().equalsIgnoreCase(type);
    type = "BOND"; // ошибка компиляции: variable used in lambda expression should be final or effectively final
}
```

Ограничение связано с внутренним представлением лямбд в JVM — подробный разбор в материале
[«Функциональная парадигма в Java»](../../materials/seminar3/ФП%20в%20Java.md).

## Функции высшего порядка

Функции, которые принимают другие функции как аргументы или возвращают их как результат. В Java роль «функции как
значения» играют функциональные интерфейсы: метод, принимающий `InstrumentFilter` или `Predicate<T>`, — это и есть
функция высшего порядка.

---

## Стандартные функциональные интерфейсы

Пакет `java.util.function` — не нужно придумывать свой интерфейс под каждую задачу:

| **Интерфейс**    | **Метод**           | **Смысл**                                     |
|------------------|---------------------|------------------------------------------------|
| `Predicate<T>`   | `boolean test(T t)` | Проверка условия — как `InstrumentFilter`     |
| `Function<T, R>` | `R apply(T t)`      | Преобразование значения одного типа в другое  |
| `Supplier<T>`    | `T get()`           | Поставщик значения без аргументов             |
| `Consumer<T>`    | `void accept(T t)`  | Действие над значением, без результата        |

`InstrumentFilter` структурно совпадает с `Predicate<Instrument>` — сигнатура `boolean test(Instrument)` вместо
`boolean matches(Instrument)`.
