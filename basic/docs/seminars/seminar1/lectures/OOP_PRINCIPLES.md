## Объект

**Объект** — сущность с состоянием и поведением. Объекты общаются через сообщения (вызовы методов).

---

## Инкапсуляция — не про геттеры и сеттеры

Инкапсуляция — это про сокрытие деталей реализации и защиту инвариантов объекта. Класс сам отвечает за то, чтобы его состояние всегда было корректным.

Пример из Си — инкапсуляция не привилегия ООП-языков:

```other
// fcntl.h — наружу торчит только контракт
int open(const char *pathname, int flags);
```

Реализация `open` разная в Linux / macOS / Windows, но клиентский код везде работает одинаково — потому что видит только контракт.

---

## Наследование

Механизм повторного использования кода и установления отношения «является» (IS-A). Имеет цену — сильную связность: изменение в базовом классе каскадно расходится по всем потомкам. Альтернативы — композиция и делегирование.

---

## Полиморфизм

Возможность работать с разными типами через общий интерфейс. Сегодня — два вида:

- **полиморфизм подтипов** (subtype) — переопределение методов, выбор реализации в runtime;
- **перегрузка** (overloading) — несколько методов с одним именем, выбор компилятором в compile-time.

---

## Как принципы дополняют друг друга

- Инкапсуляция защищает состояние.
- Наследование строит иерархии поверх защищённых объектов.
- Полиморфизм позволяет писать обобщённый код поверх этих иерархий.

Без любого из трёх — конструкция рассыпается.
