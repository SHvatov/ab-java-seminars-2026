### Темы семинара

**ООП-принципы**

- Инкапсуляция как сокрытие реализации и защита инвариантов (не просто геттеры/сеттеры)
- Наследование: IS-A, переиспользование кода, опасность каскадных изменений
- Полиморфизм подтипов (runtime) и перегрузка (compile-time)
- Взаимосвязь принципов: почему они не существуют по отдельности
- Намёк на ISP (Interface Segregation Principle) из SOLID

**Проектирование доменной модели**

- Моделирование предметной области T-Инвестиций
- Абстрактный базовый класс `Instrument` и его подклассы (`Stock`, `Bond`, `Etf`)
- Эволюция от наивной реализации (enum + static switch) к полиморфизму

**Нарушение LSP на практике**

- `getDividends(BigDecimal)` в базовом классе: почему это ошибка
- Два плохих варианта (`return ZERO` / `UnsupportedOperationException`) и правильный вывод

**Агрегация vs композиция**

- `StockSnapshot extends Stock` → проблема с перегрузкой и родительским контрактом → переименование в `Quote` и переход к агрегации
- `Portfolio` + `Position` как пример композиции

**Работа с Git**

- Клонирование репозитория, создание ветки, коммит, пуш
- Создание MR

**Числа с плавающей точкой**

- `0.1 + 0.2` и почему `double` не подходит для финансовых расчётов
- `BigDecimal`: правило создания от строки, а не от `double`

### Темы семинара

**ООП-принципы** — те же, что в базовом треке (без Git-блока, темп выше)

**Проектирование доменной модели** — та же эволюция от enum к полиморфизму

**Нарушение LSP** — тот же `getDividends` пример

**Агрегация vs композиция** — тот же `StockSnapshot` → `Quote` сценарий

**Устройство памяти в JVM**

- Heap, Non-Heap (Metaspace, Code Cache), Stack — что где живёт
- Устройство объекта: Mark Word + Class Pointer + поля + padding
- Минимальный объект — 16 байт
- Примитивы, объекты-обёртки, массивы: где хранятся в разных контекстах

**Многомерные массивы в памяти**

- Вопрос-ловушка: `int[10][1000]` vs `int[1000][10]` — что тяжелее?
- Многомерный массив = массив ссылок на массивы: почему `int[1000][10]` создаёт на 990 объектов больше

**VisualVM**

- Мониторинг Heap в реальном времени
- Наблюдение за ростом памяти при создании массивов

**JOL (Java Object Layout)**

- Подключение зависимости через Maven (`pom.xml`, Maven Central, Maven-координаты)
- `ClassLayout.parseClass()` и `ClassLayout.parseInstance()`: читаем вывод
- Понятие shallow size
- `GraphLayout` — намёк для самостоятельного изучения
