## Инкапсуляция

Не просто геттеры и сеттеры — сокрытие деталей реализации и защита инвариантов. Пример из Си: `open()` в `fcntl.h` — один контракт, разные реализации под каждую ОС. Клиентский код не знает и не должен знать, что происходит под капотом.

## Наследование

Механизм установления IS-A и переиспользования кода. Цена — жёсткая связанность, каскадные изменения в глубоких иерархиях. Краткий намёк на ISP — подробнее на семинаре 2.

## Полиморфизм

Два вида сегодня: **подтипов** (runtime, через override) и **перегрузка** (compile-time). Шаблонный (generics) — позже.

## Взаимосвязь

Полиморфизм подтипов невозможен без наследования; наследование без полиморфизма теряет смысл; инкапсуляция — фундамент, без которого всё течёт наружу.

```other
Инкапсуляция  →  сокрытие реализации, защита инвариантов
     │
     ▼
Наследование  →  построение иерархий поверх защищённых объектов
     │
     ▼
Полиморфизм   →  единообразная работа с иерархией через общий тип
```
