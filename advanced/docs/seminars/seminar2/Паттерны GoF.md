# Паттерны проектирования GoF

---

## Что такое паттерн проектирования

Типовое решение часто встречающейся проблемы проектирования — не готовый код для копирования, а шаблон, адаптируемый
под задачу. Каталог из 23 паттернов (Gamma, Helm, Johnson, Vlissides — «Gang of Four», 1994) делится на три группы по
признаку решаемой задачи: **порождающие** управляют созданием объектов, **структурные** — тем, как объекты
компонуются в более крупные структуры, **поведенческие** — тем, как объекты взаимодействуют и распределяют
ответственность.

---

## Паттерны сегодняшнего семинара

**Factory Method** (порождающий) — создание объекта, когда конкретный тип определяется динамически (строкой,
конфигурацией), без разрастания `if/else` в клиентском коде. В проекте — создание `Instrument` по строковому типу;
реализации `InstrumentFactory` обнаруживаются через `ServiceLoader`, а не самрегистрацией в статическом блоке — так
не приходится трогать `InstrumentFactories` при добавлении нового типа.

**Strategy** (поведенческий) — выбор между несколькими взаимозаменяемыми алгоритмами без условного оператора, который
пришлось бы расширять при каждом новом варианте. В проекте — фильтрация инструментов по типу через `InstrumentFilter`.

**Iterator** (поведенческий) — последовательный обход элементов коллекции без раскрытия её внутреннего устройства. В
проекте — обход `InstrumentRepository` через `Iterable`/`Iterator`, с демонстрацией `for-each` как синтаксического
сахара над этим механизмом.

**Command** (поведенческий) — запрос, инкапсулированный как объект, что позволяет параметризовать вызывающий код
разными действиями. В проекте — CLI-команды `search`/`add`/`list` поверх picocli: `@Command` + `Callable` уже
реализует паттерн на уровне библиотеки.

---

## Список паттернов GoF

**Порождающие:** Abstract Factory, Builder, Factory Method, Prototype, Singleton

**Структурные:** Adapter, Bridge, Composite, Decorator, Facade, Flyweight, Proxy

**Поведенческие:** Chain of Responsibility, Command, Interpreter, Iterator, Mediator, Memento, Observer, State,
Strategy, Template Method, Visitor
